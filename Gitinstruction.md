Git - это распределенная система управления версиями, которая позволяет эффективно отслеживать изменения в исходном коде и других файлах проекта. С помощью Git можно отслеживать, комментировать и восстанавливать различные версии проекта, а также совместно работать с другими участниками проекта. Git позволяет сохранять историю изменений, возвращаться к любому из предыдущих состояний проекта и сравнивать изменения между версиями. 

## 1. Проверка наличия установленного Git

Перед началом работы необходимо убедиться, что Git установлен на вашем компьютере. Для этого можно воспользоваться командой:

```
git --version
```
Если Git уже установлен, эта команда выведет его версию.

## 2. Установка Git

Загружаем последнюю версию Git с [официального сайта](https://git-scm.com/download/win) и устанавливаем с настройками по умолчанию.

## 3. Настройка Git

При первом использовании Git необходимо представиться. Для этого нужно ввести в терминале команды: 

Имя
```
git config --global user.name "Ваше имя"
```
E-mail
```
git config --global user.email "youremail@yourdomain.ru"
```

## 4. Инициализация репозитория

Для того, чтобы создать новый пустой репозиторий или инициализировать существующий, выполните команду:
```
git init
```
При инициализации создастся скрытая папка. В ней содержатся все объекты и ссылки, которые Git использует и создаёт в истории работы над проектом.

## 5. Запись изменений в репозиторий

### 5.1 git status

Перед записью изменений в репозиторий можно проверить, какие файлы были изменены, с помощью команды:

```
git status
```

### 5.2 git add

Чтобы добавить изменения в индекс Git, используйте команду:
```
git add <filename>
```
Замените `filename` на имя файла, который вы хотите добавить.

Если нужно добавить все измененные файлы, воспользуйтесь командой:
```
git add .
```

### 5.3 git commit

После того, как файлы добавлены в индекс Git, нужно сделать коммит, чтобы сохранить изменения в репозитории. Для этого выполните команду:
```
git commit -m "Описание изменений"
```
Замените "Описание изменений" на описание ваших изменений.

### 5.4 git diff
Для того чтобы посмотреть, какие изменения были внесены в файлы, воспользуйтесь командой:
```
git diff
```

## 6. Просмотр истории коммитов
Чтобы просмотреть историю коммитов, выполните команду:
```
git log
```
Чтобы просмотреть краткую историю коммитов, выполните команду:
```
git log --oneline
```

## 7. Перемещение между сохранениями

Если вам нужно вернуться к предыдущему сохранению, воспользуйтесь командой:
```
git checkout commit_hash
```
Замените `commit_hash` на хэш коммита, к которому вы хотите вернуться. Можно указать первые 4 символа хэша.

## 8. Игнорирование файлов

Чтобы сообщить Git какие файлы и каталоги следует игнорировать при фиксации, необходимо создать файл `.gitignore` в корневой директории репозитория.

Файл `.gitignore` может содержать различные файлы, шаблоны и правила. Например, можно использовать шаблоны, такие как (`*`) для игнорирования всех файлов или (`?`) для игнорирования файлов с одним символом в имени.

`.gitignore` может также содержать комментарии, которые начинаются с символа (`#`), чтобы помочь описать почему файл или директория должны быть игнорированы.

Как только будет создан файл `.gitignore` и добавлены нужные правила, Git будет игнорировать эти файлы и директории при отслеживании изменений в проекте.

## 9. Работа с ветками
Для просмотра списка веток в локальном репозитории используется команда:
```
git branch
```
В результате выводится список всех веток репозитория, при этом текущая ветка будет выделена звездочкой (`*`).

### 9.1 Создание ветки

Чтобы создать новую ветку, выполните команду:
```
git branch branch_name
```
, где `branch_name` - имя новой ветки.

Для того чтобы создать новую ветку и сразу переключиться на нее, используется команда
```
git checkout -b ranch_name
```

### 9.2 Переключение на ветку

Чтобы переключиться на другую ветку, выполните команду:
```
git checkout branch_name
```
, где `branch_name` - имя ветки, на которую нужно переключиться.

### 9.3 Создание и переключение на новую ветку

Для того, чтобы создать новую ветку и сразу переключиться на эту ветку, выполните команду:
``` 
git checkout -b branch_name 
```
, где `branch_name` - имя новой ветки.

### 9.4 Слияние веток

Чтобы объединить две ветки, выполните команду:
```
git merge feature_branch
```
, где `feature_branch` - имя ветки, из которой будут внесены изменения в текущую ветку.

### 9.5 Разрешение конфликтов

При слиянии веток могут возникнуть конфликты. Git покажет эти конфликты, и необходимо вручную разрешить их, редактируя соответствующие файлы.

После редактирования файла выполните команду:
```
git add <filename>
```
Замените `filename` на имя файла, который вы отредактировали, тем самым добавив новое объединенное содержимое в раздел проиндексированных файлов. 
Для завершения слияния создайте новый коммит, выполнив следующую команду:
```
git commit -m "Описание изменений"
```
Замените "Описание изменений" на описание ваших изменений.
Git обнаружит, что конфликт разрешен, и создаст новый коммит слияния для завершения процедуры слияния.

### 9.6 Удаление ветки

Перед удалением локальной ветки обязательно необходимо перейти в другую, которую не собираетесь удалять, так как нельзя удалить ветку, в которой находимся.

Чтобы удалить ветку , выполните команду:
```
git branch -d branch_name
```
, где `branch_name` - имя удаляемой ветки.

Эта команда удаляет локальную ветку, если все изменения из нее были слиты в другие ветки. Если ветка содержит изменения, которые не были слиты, то Git выведет ошибку и не удалит ветку. Если же необходимо удалить ветку с изменениями, которые еще не были слиты, то выполните команду: 
```
git branch -D branch_name
```
Эта команда принудительно удалит ветку, несмотря на наличие в ней не слитых изменений.

Кроме удаления локальных веток, можно удалить и удаленные ветки, то есть ветки, которые были удалены на удаленном репозитории. Для этого нужно выполнить команду:
```
git fetch --prune
```
Эта команда удалит все локальные ссылки на удаленные ветки. Также можно использовать команду 
```
git remote prune remote_repository_name
```
Эта команда удалит ссылки на удаленные ветки только для конкретного удаленного репозитория - `remote_repository_name`.

Для удаления веток на удаленном репозитории нужно выполнить команду 
```
git push remote_repository_name --delete branch_name
```
Эта команда удалит указанную ветку на удаленном репозитории.

Важно помнить, что удаление веток может привести к потере истории изменений. Поэтому перед удалением веток рекомендуется сохранить необходимые изменения в других ветках или создать резервную копию репозитория.

## 10. Работа с удаленными репозиториями

Работа с удаленными репозиториями в Git позволяет хранить код проекта в централизованном месте и делиться им с другими участниками команды.

### 10.1 Создание удаленного репозитория

Для начала работы с удаленным репозиторием его сначала необходимо создать на платформе для хостинга репозиториев GitHub. Для этого в правом верхнем углу веб-интерфейса нажимаем + и выбираем `New repository`. Указываем имя репозитория, описание, остальные требуемые параметры и нажимаем `Create repository`.

### 10.2 Клонирование удаленного репозитория

Для клонирования удаленного репозитория на локальный компьютер нужно выполнить команду:
```
git clone https://github.com/user/repo.git
```
, где https://github.com/user/repo.git ссылка на удаленный репозиторий.

### 10.3 Связывание локального репозитория с удаленным

Для связывания локального репозитория с удаленным необходимо выполнить команду:
```
git remote add origin https://github.com/user/repo.git
```

### 10.4 Отправка изменений в удаленный репозиторий

Для отправки изменений проекта в удаленный репозиторий нужно выполнить команду:
```
git push -u remote_name branch_name
```
, указав имя удаленного репозитория `remote_name` и имя ветки `branch_name`, в которой находятся изменения. Например:
```s
git push -u origin main
```
Эта команда отправляет изменения, сделанные в локальной ветке `main`, на удаленный репозиторий `origin` в ветку `main` и устанавливает отслеживание этой удаленной ветки.

После установки отслеживания, можно использовать команду 
```
git push 
```
без указания имени ветки и удаленного репозитория.

### 10.5 Проверка состояния удаленного репозитория
Чтобы просмотреть список удаленных репозиториев, связанных с текущим локальным репозиторием, и их URL нужно выполнить команду:
```
git remote -v
```

### 10.6 Получение изменений из удаленного репозитория
Эта команда не сливает изменения в текущую рабочую ветку и не изменяет локальный репозиторий, но только загружает изменения из удаленного репозитория в локальный репозиторий.


Чтобы загрузить изменения из указанного удаленного репозитория в локальный репозиторий, но не применяет их к текущей ветке, необходимо выполнить команду:
```
git fetch origin
```
Эта команда полезна, когда нужно проверить наличие обновлений из удаленного репозитория, не интегрируя их в локальную кодовую базу.
Для того чтобы слить изменения из удаленной ветки используют команду:
```
git merge origin/main
```
Для получения изменений из удаленного репозитория и слияния их с текущей веткой используется команда:
```
git pull remote branch
```
Эта команда загружает изменения из указанной ветки удаленного репозитория и сливает их с текущей локальной веткой, объединяя команды `git fetch` и `git merge`.

### 10.7 Удаление удаленных веток и репозиториев
Для удаления ветки на удаленном репозитории вы можете воспользоваться командой 
```
git push origin --delete branch
```
Эта команда удалит указанную ветку из удаленного репозитория.

Для удаления удаленного репозитория с сервера GitHub можно воспользоваться командой 
```
git remote rm <имя репозитория>
```
Эта команда удалит ссылку на удаленный репозиторий, но не удалит сам репозиторий с сервера.
